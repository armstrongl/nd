## Update docs to be agent-agnostic

### Objective

Replace all Claude Code-specific references in `docs/` and `README.md` with generic, agent-agnostic terminology so the documentation works for any coding agent (Claude Code, Cursor, Windsurf, Copilot, etc.). There are approximately 150 occurrences across 52 files in `docs/` plus 3 in `README.md`.

### Tasks

- [ ] Audit all `docs/guide/` files for "Claude Code" references and replace with "your coding agent" or "the agent" as appropriate
- [ ] Audit all `docs/reference/` files (auto-generated by gendocs) and update the gendocs templates/source strings so regenerated docs are also agent-agnostic
- [ ] Replace hardcoded `.claude/` paths with a generic placeholder like `<config-dir>/` or the nd config directory path, adding a note that the actual path depends on the agent
- [ ] Update `docs/guide/glossary.md` (14 occurrences): redefine agent-specific terms generically, add a "Supported agents" entry listing known-compatible agents
- [ ] Update `docs/guide/how-nd-works.md` (21 occurrences): rewrite the architecture description to reference "agent config directory" instead of `.claude/`
- [ ] Update `docs/guide/troubleshooting.md` (11 occurrences): replace agent-specific diagnostic steps with generic equivalents
- [ ] Update `docs/guide/configuration.md` (7 occurrences): use `nd` config paths, not `.claude/` paths
- [ ] Update all `docs/guide/asset-types/*.md` files: replace agent-specific deployment paths and examples
- [ ] Update `README.md`: replace any Claude Code branding with generic agent language
- [ ] Search for `claude-code` (kebab-case) references in docs and replace
- [ ] Run `scripts/lint-docs.sh` to verify documentation style compliance after changes
- [ ] Verify no broken internal links after the rename (check cross-references between guide pages)

### Acceptance criteria

- `grep -ri "claude code" docs/ README.md` returns zero matches (exception: a "Supported agents" list that mentions Claude Code as one of several agents is acceptable)
- `grep -r "\.claude/" docs/ README.md` returns zero matches outside of explicit "agent-specific path" callout boxes
- `scripts/lint-docs.sh` passes with no errors
- `docs/guide/glossary.md` defines terms generically and lists supported agents
- Auto-generated reference docs (`docs/reference/`) are also agent-agnostic after running `go run cmd/gendocs/main.go`
- All internal doc links resolve correctly (no broken `[text](path)` references)
